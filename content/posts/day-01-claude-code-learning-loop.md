今天的目标很简单：

1. 快速通读 Claude Code 官方文档，建立能力地图。
2. 选 1 个最常用场景立刻实践。
3. 把结果写成可复用模板，明天直接套用。

## 我今天做了什么

### 1) 建立能力地图（20 分钟）

我按目录扫了一遍文档，只记录了三类信息：

- 高频：每天都会用到
- 中频：每周会用到
- 低频：先知道有这个能力

### 2) 实践一个真实任务（30 分钟）

场景：初始化一个新的博客仓库并接入已有主题。

我把目标直接写给 Claude Code：

- 初始化 Hugo 站点
- 接入主题子模块
- 写一篇示例文章
- 配置 GitHub Pages 自动部署

### 3) 沉淀模板（10 分钟）

我把今天有效的提示结构固定下来，后续复用：

```text
你是我的工程协作助手，请按下面步骤执行：
1) 先确认当前目录状态
2) 以最小可运行方案初始化项目
3) 每完成一步给出可验证结果
4) 遇到阻塞先给替代路径，不要停
```

## 今日结论

- “先通读再深挖”比直接啃细节更高效。
- 文档学习一定要绑定真实任务，否则记不住。
- 把提示词做成模板后，执行稳定性明显更高。
